# 2026 Onsite Site Build Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Revive the parameterized onsite flavor of the nix.vegas site (ISOs/netboot/docs rendered on `/2026/onsite`) and deploy it to ghostgate, while the default no-artifact build keeps serving Netlify/crystal, per `docs/superpowers/specs/2026-07-16-onsite-site-2026-design.md`.

**Architecture:** In the site repo, `pkgs/nix-vegas-site/onsite.nix` (restored from `4deb0cb^`, parameterized with `baseUrl`) overrides the zola build: tomlq rewrites `config.toml` `[extra]` values discovered from an `onboardingArtifacts` derivation, which is also symlinked into `public/nixos`. Shortcodes gain empty-value defaults so the offsite render looks intentional. In the systems repo, an overlay applies `.override { onboardingArtifacts = nixos-lv-onboarding-artifacts; }` and ghostgate serves it with `/` 302-ing to `/2026/onsite`.

**Tech Stack:** zola (Tera templates/shortcodes), tomlq (yq)/jq, Nix flakes, nginx, deploy-rs.

## Global Constraints

- Two working trees: site = `~/projects/nix.vegas` (branch `onsite-2026`, user pushes), systems = `/home/numinit/systems` (current branch).
- **Commits unsigned in-session**: always `git commit --no-gpg-sign` (YubiKey signing hangs; user re-signs).
- Names are load-bearing: flake outputs `nixVegasOnsite`/`nixVegasOffsite`, overlay attr `nix-vegas-site-onsite`, artifacts attr `nixos-lv-onboarding-artifacts`, redirect target `/2026/onsite`.
- The **default site build must not change behavior** except for empty-value shortcode renders (Netlify/crystal consume it).
- Empty-value shortcode copy, verbatim: lists/blocks → `<em>Available on the event network.</em>`; inline values with no better text → `(onsite)`.
- Systems-side builds before the site branch is on GitHub must use `--override-input nix-vegas-site "git+file://$HOME/projects/nix.vegas?ref=onsite-2026"`.

- [ ] **Step 2: Write `pkgs/nix-vegas-site/onsite.nix`**

The 2025 file, with `base_url` parameterized:

```nix
{
  nix-vegas-site,
  jq,
  yq,
  emptyDirectory,
  onboardingArtifacts ? emptyDirectory,
  baseUrl ? "https://nixos.lv/",
  ...
}:

nix-vegas-site.overrideAttrs (prev: {
  inherit onboardingArtifacts;
  nativeBuildInputs = prev.nativeBuildInputs or [ ] ++ [
    yq
    jq
  ];
  preBuild = ''
    slurp_jq() {
      jq --raw-input --slurp '(split("\n") | map(select(. != "")))'"$*"
    }
    ln -s $onboardingArtifacts nixos
    tomlq -ti '.base_url = "${baseUrl}"' config.toml
    tomlq -ti '.extra.onsite = true' config.toml
    tomlq -ti ".extra.nixpkgs_rev = $({ cat nixos/rev || true; } | slurp_jq '[0] // ""')" config.toml
    tomlq -ti ".extra.nixos_version = $({ cat nixos/version || true; } | slurp_jq '[0] // ""')" config.toml
    tomlq -ti ".extra.isos = $(
      { find -L nixos/systems/ -name '*.iso' || true; } | slurp_jq
    )" config.toml
    tomlq -ti ".extra.vmas = $(
      { find -L nixos/systems/ -name '*.vma.zst' || true; } | slurp_jq
    )" config.toml
    tomlq -ti ".extra.manual = $(
      { find -L nixos/manual/ -name index.html || true; } | slurp_jq '[0] // ""'
    )" config.toml
    tomlq -ti ".extra.search = $(
      { find -L nixos/search/ -name index.html || true; } | slurp_jq '[0] // ""'
    )" config.toml
    tomlq -ti ".extra.channel = $(
      { find -L nixos/channel/ -name '*.tar.xz' || true; } | slurp_jq '[0] // ""'
    )" config.toml
    rm -f nixos

    echo "Final config.toml:" >&2
    cat config.toml >&2
  '';

  postInstall = ''
    ln -s $onboardingArtifacts $out/public/nixos
  '';
})
```

### Downloading packages from our binary cache

Our local binary cache is at [https://cache.nixos.lv](https://cache.nixos.lv). It serves
everything we've built and fetched for the event, straight from our store over the
event network. If you're using our channel:

`nix-shell --option substituters https://cache.nixos.lv -p ghidra`

Heading home? These names are event-only: offsite they just redirect to nix.vegas, and your
Nix will warn and fall back to [https://cache.nixos.org](https://cache.nixos.org) on its own.

when you get home to skip the warning.

our local [nixos-pagefind](https://github.com/Jaculabilis/nixos-pagefind) built in the
